POST    /switches

Create a switch by providing a switch IP address and associated credentials.  The POST action shall trigger switch polling. During the polling process, MAC address of the devices connected to the switch will be learned by SNMP or SSH.

Normal Response Code: 202

Error Response Codes: ObjectDuplicateError(409), UserInvalidUsage(400)

Request parameters:

Parameter

Type

Description

ip

string

The switch IP address.

version

string

SNMP version when accessing the specified switch by SNMP.

community

string

SNMP community when accessing the specified switch by SNMP.

username

string

SSH username when accessing the specified switch by SSH.

password

string

SSH password when accessing the specified switch by SSH.

Response parameters:

Parameter

Type

Description

status

string

“accepted” if the switch is created successfully, or appropriate error string in case of failure.

state

string

Valid only if the switch is created successfully. Once polling switch is completed and MAC addresses retrieved, the state of the switch shall be “under_monitoring”, otherwise it shall be “not_reached”.

id

int

The unique identifier of the switch

link

dict

The switch link

Error Response parameters:

Parameter

Type

Description

status

string

Brief error description

message

string

The details description for the error

failedSwitch

string

The ID of the failed switch

{

    “switch”: {

        "ip" : "192.168.10.2",

        "credential" : {

                “version”: “v2c”,

        }

    }

}

        "ip" :, "192.168.10.2"

POST    /clusters/{id}/action

Takes actions for the specified cluster. The actions includes addHosts, removeHosts, replaceAllHosts, and deploy. The action “addHosts” allows a user to add the specified machine(s) as the host(s) to the cluster. The action “removeHosts” allows to remove the specified the host(s) by host ID(s) from the cluster. The action “replaceAllHosts” allows to remove all existing hosts of the cluster and add new machine(s) as the new host(s) to the cluster. The action “deploy” shall deploy and start to install each host(s) in the cluster.

Normal Response Code: 200

Error Response Codes: UserInvalidUsage(400), ObjectDoesNotExist(404), ObjectDuplicateError(409)

Request parameters:

Parameter

Type

Description

id

int

The unique identifier of the cluster

addHosts

list

The list of machine ID(s) expected to be added as the host(s) to the cluster.

removeHosts

list

The list of host ID(s) expected to be removed from the cluster.

replaceAllHosts

list

The list of machine ID(s) expected to be added as the new host(s) to the cluster and replaced the existing hosts in the same cluster.

deploy

dict

None dict to trigger cluster deploy action.

Response parameters:

Parameter

Type

Description

status

string

“OK” if no error occurs, or appropriate error string in case of failure.

id

int

The unique identifier of the cluster host

machine_id

int

The machine ID

deployment

string

The link of the installing progress of the specified cluster.
